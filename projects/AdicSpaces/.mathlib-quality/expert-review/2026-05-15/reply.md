The main issues are:

1. **W1 is plausible but may be too absolute.** For a cover of an arbitrary rational base `C.base`, the standard cover should morally be a standard cover over `O(C.base)`, not necessarily a finite `S ⊆ A` with `Ideal.span S = ⊤` in `A`. If the project already has a theorem turning relative standard covers into `A`-labelled plus-pieces, then W1 is fine. Otherwise W1 should be relativised.

2. **W2 is missing the essential output.** It should not merely say that some subfamily `I_units(L)` consists of units on each leaf. That is vacuous if `I_units = ∅`. Wedhorn's Step (ii) says: on each first-stage Laurent leaf `L`, the **restriction of the standard cover to `L` is generated by units**. So W2 must output a unit family that actually covers/refines the restricted standard cover.

3. **W2 should include the Cor. 7.32 dominating-unit consequence.** The existential `s` is not enough unless the statement includes the derived domination/covering property. Choosing `s = 1` can prove some "minus indices are units" facts, but it does not prove the Wedhorn Step (ii) covering statement.

4. **W3 is too strong and probably has the wrong type.** Unit ratios live in `O(L)`, not generally in `A`. Returning a plain `LaurentTree A` is not mathematically justified unless you have a precise transport theorem from relative ratio splits over `O(L)` to absolute `A`-labelled rational data.

5. **W4 is false as stated.** Arbitrary `inner_of` functions can produce coinciding leaves at distinct outer bases. Cross-leaf disjointness is not a mathematical theorem of Wedhorn; it is a Finset bookkeeping artifact. It should be handled by indexed leaves, pruning, or an explicit pairwise-disjointness hypothesis for the canonical construction.

So the composition W1 → W2 → W3 → W4 → W5 is *morally* the right plan, but the current W2, W3, and W4 statements should be revised before proof attempts.

## Mathematical idea

Wedhorn's construction has this quantifier structure:

```
Given a standard cover U generated by f₀,…,fₙ,
choose a dominating unit s by Cor. 7.32.
Let V be the Laurent cover generated by s⁻¹fᵢ.
For every first-stage Laurent leaf L of V,
  the restricted cover U|L is generated by units in O(L).
Then every rational cover generated by units
  is refined by a Laurent cover generated by pairwise ratios of those units.
Therefore after grafting these ratio-Laurent refinements under each L,
  the final leaves refine U, hence refine the original cover C.
```

The crucial point is that the first-stage Laurent leaves do **not** themselves refine `C` leaf-by-leaf. The all-minus leaf is not a problem because it is not terminal. It is further refined by the unit-ratio Laurent cover attached to that leaf.

So the correct intermediate object is not:

```
first-stage leaf L ⊆ one C-piece
```

but rather:

```
standard cover restricted to L is generated by units,
and that unit-generated cover is refined by a ratio Laurent cover.
```

This also answers the "generated by units" ambiguity. It means:

```
For each first-stage leaf L,
there is a finite family of units u₀,…,u_m ∈ O(L)
such that the restricted standard cover U|L
is the rational cover generated by those units.
```

It does **not** mean each original `f_i` is a unit on each `C`-piece, and it does **not** mean the restricted cover is automatically the singleton cover. Wedhorn's Step (iii) is needed exactly because a unit-generated rational cover still needs a Laurent-ratio refinement.

## Risks or missing facts

The largest risk is proving vacuous or too-strong W2/W3 statements. If W2 only says "there exists a unit subfamily," the later W3 has to prove impossible facts from insufficient input. Add the restricted-cover-generated-by-units condition now.

The second risk is forcing ratio Laurent covers into `LaurentTree A`. Ratios live in `O(L)`. Unless you formalise a transport theorem, an `A`-labelled tree is not the right object for the inner refinement.

The third risk is disjointness. Wedhorn's proof is about covers, where duplicate pieces are harmless. Your Finset-based tree proof needs disjointness only for product homeomorphism bookkeeping. Handle it by indexing or pruning, not by asserting false cross-leaf disjointness.

The fourth risk is hidden final-hypothesis pollution. W2 should consume Cor. 7.32 internally and output the clean derived unit-generated-cover property, rather than pushing Cor. 7.32's technical side conditions all the way to `IsSheafy`.

The fifth risk is W1's absolute span-top condition. If the cover is of an arbitrary rational base, verify that `refines_span_top S` in `A` is really what your Hübner/Nullstellensatz refinement theorem produces. Otherwise make W1 relative over `O(C.base)`.
